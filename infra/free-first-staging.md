# KRYNO Free-First Staging Path

This is the staging path we will use before spending money on infrastructure. The rule is simple: use free tiers until a real product limit blocks testing, then upgrade only that service.

## Free Accounts To Create Now

| Need | Recommended Free Provider | Why We Need It | Required Value For KRYNO |
| --- | --- | --- | --- |
| DNS/CDN/WAF | Cloudflare Free | Public HTTPS domain, proxy, basic protection | Domain zone, optional API token |
| Database | Supabase Free | Hosted Postgres for auth, posts, chat metadata | `DATABASE_URL` |
| Redis | Upstash Free | Distributed rate limits and abuse protection | `REDIS_URL` |
| Monitoring | Sentry Free | Backend and mobile crash/error reports | Backend `SENTRY_DSN`, mobile `EXPO_PUBLIC_SENTRY_DSN` |
| Email | Existing free SMTP or Resend Free | Verification, reset, security alerts | SMTP host, port, user, pass, from email |
| Calls | LiveKit Free | Real audio/video staging tests | `LIVEKIT_URL`, API key, API secret |
| Media storage | Cloudflare R2 Free allowance or Supabase Storage Free | Profile photos, posts, stories | R2/S3 keys or Supabase storage decision |

## Not Required Immediately

- Cloudflare Pro is not required for staging.
- Supabase Pro is not required for early staging.
- Paid Redis is not required until limits are hit.
- Paid Sentry is not required until event volume/team workflow needs it.
- Paid email is not required if verification/reset emails arrive reliably.
- Paid TURN is not required for the first LiveKit staging test, but should be planned before public launch if call reliability is weak.

## Must Pay Later

| Item | When It Becomes Required |
| --- | --- |
| Domain name | Before permanent staging/production branding |
| Google Play Console | Before Play Store release |
| Apple Developer Program | Before iOS TestFlight/App Store release |
| Cloudflare Pro or stronger WAF | Before paid public launch or high-risk traffic |
| Supabase Pro/backups | Before paid public launch with real user data |
| Paid email provider | Before paid launch if free SMTP delivery is unreliable |
| Dedicated TURN/call infra | Before launch if two-device call tests fail or become unstable |

## Setup Order

1. Create Supabase Free project and copy the pooled Postgres connection string. Status: complete, migrations applied.
2. Create Upstash Redis Free database and copy the TLS Redis URL. Status: complete, Redis ping verified with local Node system CA.
3. Create Sentry Free projects: one backend, one mobile. Status: complete, DSNs collected.
4. Confirm the free SMTP provider can send to Gmail without landing in spam. Status: SMTP login verified with `verify.kryno@gmail.com`; local PC required `SMTP_TLS_REJECT_UNAUTHORIZED=false` for certificate-chain compatibility.
5. Create LiveKit Free project and copy server credentials. Status: complete, token generation verified locally.
6. Decide media storage: Cloudflare R2 is preferred if we already use Cloudflare; Supabase Storage is acceptable for earliest staging. Status: R2 bucket created, scoped read/write token created, upload/head/delete verified locally, public development URL enabled.
7. Fill backend host secrets.
8. Run backend migration.
9. Run staging security check.
10. Build the Android APK/AAB against the staging API URL.

## Free-First Exit Criteria

We stay on free tiers while:

- sign up, email verification, login, refresh, and logout work;
- profile photo, story, post image upload, and likes work;
- chat works on two devices;
- calls connect on two devices during normal testing;
- Sentry receives errors without PII;
- rate limiting works through Redis;
- no provider free limit is blocking realistic testing.

We upgrade only when one of those checks fails because of a free-tier limit or reliability issue.
